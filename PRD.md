# PRD: AI 评论分析 Agent

## 1. 产品定位

**一句话描述：** 上传 CSV → 自动分析 → 可追问的报告——海量用户反馈的 AI 分析工具。

**Slogan（待定）：** 高效地从用户声音中发现产品机会。

**产品形态：** 轻量级 Web 工具（Streamlit），本地运行，非注册制 SAAS 平台。

**当前阶段：** MVP（最小可用产品），个人使用 + 面试作品展示。

## 2. 用户与场景

| 维度 | 内容 |
|------|------|
| **核心用户** | 产品经理、运营人员 |
| **使用场景** | App/平台发布新版后，大量用户评论涌入，需要快速了解"用户在吐槽什么"并决定优化优先级 |
| **使用频率** | 按需使用（数据更新时触发分析），一次分析 5-15 分钟 |
| **用户痛点** | ① 手工看评论效率低 ② 通用 LLM 上下文不够 ③ 缺乏从洞察到决策的追溯链路 |

## 3. 市场分析

### 3.1 竞品分析

| 竞品层级 | 典型方案 | 优势 | 劣势 |
|----------|----------|------|------|
| **手工+通用工具** | Excel 统计、Python 脚本 | 灵活、免费 | 耗时长，无自动总结 |
| **通用 LLM** | ChatGPT、DeepSeek 对话 | 零门槛上手 | 上下文窗口有限，无追溯，无 pipeline 自动化 |
| **专业分析平台** | Sprout Social、Brandwatch | 功能完善、可视化强 | 价格高、部署重、不可本地化 |
| **开源方案** | 自建 BERT+聚类 | 完全可控 | 技术门槛高，非技术用户不可用 |

### 3.2 差异化定位

本产品位于**通用 LLM 与专业平台之间**：
- 比 ChatGPT 处理量大（万级评论无压力）、分析可追溯（Dashboard→类别→子问题→原始评论）
- 比专业平台轻量、免费、本地部署、数据不出本机
- Agent 决策链（ReAct）而非单次 prompt——LLM 自主决定查什么数据、对比什么维度

### 3.3 核心价值

| 价值点 | 说明 |
|--------|------|
| **效率** | 10000 条评论自动分类+总结，5-15 分钟出报告 |
| **可追溯** | 每条结论能下钻到原始评论，不是黑盒总结 |
| **Agent 驱动** | LLM 自主规划分析策略+质量自检+追问交互，非固定 pipeline |
| **低成本** | DeepSeek-chat 模型，单次分析成本极低（<0.1 元） |

## 5. 技术架构

```
用户上传 CSV
      ↓
[Agent Plan]   检查数据 → 决定参数 → 展示计划 → 用户确认
      ↓
[Pipeline]     Step1 清洗+向量化 → [Checkpoint] 质量检查
               Step3 分类+LLM分析 → [Checkpoint] 置信度检查+自动重试
      ↓
[Agent Review] LLM 自检每子问题 → 置信度标签
      ↓
[Streamlit]    Tab1 上传与规划 / Tab2 报告+图表 / Tab3 ReAct 追问
```

- **前端:** Streamlit + Altair 图表
- **LLM:** DeepSeek-chat（OpenAI 兼容 API）
- **Embedding:** SentenceTransformer all-MiniLM-L6-v2
- **分类:** 关键词匹配 + LLM 子问题拆分

## 6. 关键指标（评价标准）

| 指标 | 目标 | 测量方式 |
|------|------|----------|
| 分析质量 | 子问题置信度 low 占比 < 10% | Agent Review 统计 |
| 处理能力 | 10000 条评论 15 分钟内完成 | Pipeline 计时 |
| 追问准确性 | LLM 回答引用数据准确率 > 90% | 人工抽查 |
| UI 可用性 | 用户首次使用无需指导即可完成分析 | 可用性测试 |

## 7. 已知风险与缓解

| 风险 | 严重度 | 缓解措施 |
|------|--------|----------|
| LLM 幻觉（分析结果不准确） | 高 | ① Agent Review 自检+置信度标签 ② 低置信度自动重试 ③ 每条结论可追溯原始评论 |
| 状态感知不足 | 中 | ① Pipeline 检查点可见 ② 各阶段状态提示 ③ 错误信息友好展示 |
| 网络依赖（HuggingFace/CDN） | 中 | HF_HUB_OFFLINE 离线模式、本地模型缓存 |
| 数据安全（用户上传敏感评论） | 低 | 全本地运行，数据不出本机 |
